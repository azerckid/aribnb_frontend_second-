# 백엔드 수정 제안 검토 결과

## ✅ 수정 진행해도 됩니다!

프론트엔드 코드를 확인한 결과, 백엔드의 수정 제안이 모두 적절합니다.

## 검토 결과

### 1. ✅ `beds` 필드 - **수정 필요**

**현재 상황:**
- 프론트엔드: 업로드 폼에서 `beds`를 **number (침대 개수)**로 입력받고 있음
- 프론트엔드 타입: `beds: unknown[]` (잘못된 타입) → **이미 `beds: number`로 수정 완료**
- 백엔드: Bed 모델로 개별 침대 관리 중

**결론:**
- ✅ 백엔드에서 `beds` 필드를 **number (침대 개수)**로 추가하는 것이 맞습니다
- 프론트엔드 타입은 이미 수정 완료했습니다

### 2. ✅ 응답 상태 코드 201 Created - **수정 권장**

**현재 상황:**
- 프론트엔드: 200 OK도 201 Created도 모두 처리 가능
- RESTful API 관례: 리소스 생성 시 201 Created 사용

**결론:**
- ✅ 201 Created로 변경하는 것이 RESTful API 관례에 맞습니다
- 프론트엔드에는 영향 없음

### 3. ✅ `id`와 `pk` 명시 - **수정 권장**

**현재 상황:**
- 프론트엔드: `id || room.pk || 0` 형태로 둘 다 사용 중
- 타입 정의: `id: number`, `pk?: number` 둘 다 포함

**결론:**
- ✅ Serializer에서 `id`와 `pk`를 명시적으로 포함하는 것이 좋습니다
- 프론트엔드 호환성 유지에 도움됨
